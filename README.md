# 🛡️ Fintech-Agent: 지능형 사기 탐지 및 자율 대응 에이전트

본 프로젝트는 **Kaggle PaySim** 데이터를 활용하여 금융 사기 거래를 탐지하고, AI 에이전트가 실시간으로 사용자에게 대응하는 시스템을 구축하는 캡스톤 디자인 프로젝트입니다.

---

## 📊 데이터 분석 및 인사이트 (EDA)
최근 수행된 데이터 탐색 분석(EDA)을 통해 발견한 주요 특징은 다음과 같습니다.

### 1. 데이터 특성
* **데이터 규모:** 총 6,362,620건 (No Missing Values)
* **극심한 클래스 불균형:** 사기 거래 비율은 전체의 **0.1291% (8,213건)**에 불과함.
  * *대응 전략:* 모델 학습 시 SMOTE 오버샘플링 및 트리 기반 모델(XGBoost 등) 활용 예정.

### 2. 주요 사기 패턴 (Risk Vectors)
* **집중 경로:** 사기 거래는 100% **TRANSFER(송금)** 및 **CASH_OUT(인출)** 유형에서만 발생.
* **이상 징후:** 거래 금액이 `0`인 비정상 거래(16건) 발견, 시스템 테스트 및 사기 징후로 판단됨.
* **선형성:** 단순 수치(금액 등)와 사기 여부의 선형 상관관계는 낮음 → 비선형 관계를 파악하는 딥러닝/머신러닝 모델 필요.

## 🛠️ 시스템 아키텍처 및 역할 분담

### 🧑‍💻 데이터 분석 (Byeongchan)
- **탐색적 데이터 분석(EDA):** 데이터 불균형 확인 및 사기 거래 특징 추출.
- **피처 엔지니어링:** 잔액 차이 분석, 계좌 간 거래 패턴 수치화.
- **예측 모델링:** XGBoost를 활용한 실시간 사기 확률(Probability) 산출.

### 🤖 AI 에이전트 (Seungbin)
- **자율 대응 로직:** 분석 모델이 전달한 위험 점수를 바탕으로 의사결정 수행.
- **사용자 인터랙션:** 위험 거래 발생 시 사용자에게 대화형 확인 메시지 발송.
- **실행 자동화:** 사용자 응답에 따른 거래 승인/차단 자동 실행 엔진 구축.
